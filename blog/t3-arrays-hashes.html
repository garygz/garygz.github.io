Arrays and Hashes in Ruby
Usage samples of different collections
11/13/2014
Array and Hash maps are collection classes in Ruby that allow to store lists of objects. Array is an ordered list of objects, while a Hash is an unordered list of key value pairs. The key difference is a look up time of an element and memory consumption. Array elements can be accessed by using an index which allows for afst random access of an element. However, if there is a need to look for a particular element by an object attribute then Hash class is a much better collection type since it keeps key/value pairs. Iteration over an array is faster. Arrays also require less memory to store the data. These are a few way to create a new Array or Hash in ruby: 

    #create a new empty array
    array = []
    #create a new array containing these numbers 1,2,3
    array = [1,2,3]
    #create a new Hash
    hash = {}
    #create a new Hash with product/price key/value pairs
    hash = {:milk=>1.5, apples=>1.25, oranges=>2.3}
This is how to create an array of predefined size of 10: 

    array = Array.new(10)
This is how to create an array of a pre–defined size of 10 with all values set to a default value of 100: 

    array = Array.new(10,100)
Array elements are accessed by using square brackets - [n], at or a fetch method. Indexes are zero based meaning the first element has an index of 0. 

      array = [1,2,3]
      value = array[0]
      puts value == 1 #this will return true

      value = array.at(1)
      puts value == 2 #this will return true


      #This will raise an exception since
      # there is no element with an index of 4.
      #This how fetch differs from [] or at
      #method which will return nil
      value = array.fetch(4)

More information about Ruby arrays can be found here: Arrays

Hash values get accessed by using a square bracket or a fetch method with the key. Unlike in the Array class, fetch method will not raise an exception but will return a nil if the key does not exist. It also supports one more argument which specifies the default value to return. 

      #create a new Hash with product/price key/value pairs
      hash = {:milk=>1.5, apples=>1.25, oranges=>2.3}

      value = hash[:milk]
      puts 1.5 == value #this is true

      value = hash.fetch(:milk)
      puts 1.5 == value #this is true

      #return the default value for a non-existant key
      value = hash.fetch(:cake, "not available");
      puts "not available" == value #true

More information about Ruby hashes can be found here: Hashes
