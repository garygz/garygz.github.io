Regular Expression
A powerful way to search and parse strings
12/21/2014

Regular expressions is a powerful string search and parsing notation, usually abbreviated as regex or regexp. It allows to define a concise pattern search which can be used to replace a number of characters in a text string that will match the pre-defined criteria in the regular expressions. There are character classes, meta-characters, quantifiers and ranges in regular expressions which can be combined into more complex structures.

Let's see regex in action:


  (Test_String) regex expression will find the pattern defined in parenthesis.
  So for this sample string "This is a sample Test_String", "Test_String" will be returned.

  [A-Z] is a range that will look for all upper case letters from A to Z.

  [AB] matches either an upper case A or B.

The table below demonstrates some of the character classes and meta-characters and explains how to use them:

Regular Expression Type Description
+ Quantifier Match at least one or more characters that are followed by +.
* Quantifier Match zero or more characters that are followed by *.
. Quantifier Any character except new line (\n). This regex: .+ will return the entire string "test 1627 string"
\w Character Class Alphanumeric characters plus "_". This regex: \w+ will match "test" in "test string". Since there is white space separating the two words only the first word will match \w+
\d Character Class Any digit. This is the same as [0-9]. \d+ will match 1627 in "test 1627 string"
\s Character Class White space. This is very useful to break a sentence into an array of words (e.g. "test this string".split(/\s/) will return %w[test this string]
