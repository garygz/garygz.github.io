Cascading Style Sheets (CSS)

CSS best practices

11/9/2014

Cascading Style Sheets (CSS) is a language used to format and define the look and feel of HTML pages. It is the best practice to create a separate CSS file and import it into the web page rather then define CSS rules in the HTML page itself. The reason for it that many elements of the HTML page such as pages, buttons, navigation bars should look similar. Using a separate file allows to reuse the same CSS definitions and reduce the size of the HTML pages that need to be loaded. On top of that CSS files can be cached by browsers allowing for even faster loading times.

CSS language allows to define styles based on element type classes and individual tag ids which further allows to customize the elements in each HTML page.

Media queries are special feature of CSS that allows to define different layout rules for different device types (e.g. mobile devices vs laptops).

Responsive design relies a lot on media queries as the main principle of responsive design is to create CCS style rules taht can properly display the web-page on all types of devices.

There is a special sense of satisfaction from being able to write an entire site form scratch without using professional templates. It requires a set of skills such as deep understanding of CSS, a sense of design and coding in javascript.
