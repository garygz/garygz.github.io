Ruby vs JavaScript

Differences in looping

12/13/2014

While there are a lot of similarities between Ruby in JavaScript, looping is probably the prime example of where Ruby offers a lot of functionality out of the box compared to JavaScript.
While both languages support while and for loops, Ruby offers a bunch of iterator methods which minimize the amount of code required to map, select, delete or aggregate data on both maps and arrays. Maps and arrays in Ruby both implement the same interface Enumerable and therefore offer the same kind of iteration functionality. In JavaScript, however, there are no such methods provided by arrays or JSON objects. In order to support a function has to be introduced that can provide the functionality. While it is possible to create functions in JavaScript for map, select and other Ruby like methods, programmers either have to use a global common library or create those functions in place.

Let's take a look at a function that doubles values in a Ruby map vs JavaScript object: 


      hashMap = {:value1=>10, :value2=>2, :value=>15}

      hashMap.update(hashMap){ |k,v| v*2 }

      p hashMap #<={:value1=>20, :value2=>4, :value=>30}

Here is the same basic code in JavaScript


      map = {
      value1:10,
      value2:2,
      value3:15

      }

      for (var key in map) {
      map[key] = map[key]*2;
      }
      console.log(map);//{ value1: 20, value2: 4, value3: 30 }

The code below achieves the same results using Ruby like implementation:


      var mapFunction = function(array, func) {
      var results = {};

      for (var key in map) {
      results[key] = (func(map[key]));
      }

      return results;
      };


      map = {
      value1:10,
      value2:2,
      value3:15

      }

      var doubleValueFunction = function(x) { return x * 2 };

      var doubleMap = mapFunction(map, doubleValueFunction);
      console.log(doubleMap);//{ value1: 20, value2: 4, value3: 30 }

As discussed above, Ruby code is a lot more compact and only requires one line of code to implement the logic. However, with the new standard called ECMAScript 6 the differences will be minimized.
